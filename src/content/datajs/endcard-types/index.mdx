# End Card / Playable Shell Types

These data types control the playable's shell — preloader, version selector, custom fields and the Interactive End Card (IEC).

- [Preloader](endcard-types/preloaderdata.md)
- [Version](endcard-types/versiondata.md)
- [CustomFields](endcard-types/customfieldsdata.md)
- [IEC (Legacy)](endcard-types/iecdata.md)
- [IEC2](endcard-types/iec2data.md)

## Preloader

The `Preloader` data type configures the playable's loading screen (logo, background pattern, progress bar, text).

[Learn more about Preloader Data](endcard-types/preloaderdata.md)

## Version

The `Version` data type lets the user pick a numeric variant/version of the playable.

[Learn more about Version Data](endcard-types/versiondata.md)

## CustomFields

The `CustomFields` data type exposes dynamic, user-defined fields scoped to a specific version (name, key, type, value, required).

[Learn more about CustomFields Data](endcard-types/customfieldsdata.md)

## IEC

The legacy Interactive End Card. Configures the post-play screen with background, logo, CTA, retry button and text styling.

[Learn more about IEC Data](endcard-types/iecdata.md)

## IEC2

The next-generation Interactive End Card with item-based body and multiple body variants (standard, random, timed, sequence). Prefer this for any new playable.

[Learn more about IEC2 Data](endcard-types/iec2data.md)
